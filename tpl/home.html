This is the place where the best work of mine can be seen. It is good for inspiration or for production use as well. It aims to creation of application's administration (frontend and backend too). It uses AngularJS + jQuery and J2EE + Jersey.

Main features

Scalability (lazy loading)
As administration interface gets bigger you don't have to worry about scalability, most content is downloaded lazyli on demand which saves server's and client's resources too.
Entity management setup in minutes

Application uses miloTable heavyweight directive which is highly customizable and with right REST endpoint it will do 90% of work when you do:

  • Entities preview,
  • filtering,
  • sorting,
  • create,
  • update,
  • or delete.
Stateless
Altough this is fully one-page Javascript application it holds important state in URL so you can reload anytime and stay in same state. Nice thing is that there are always loaded only needed sources.
Routing conventions
Makes creating new pages very fast
Easy localization
Easy CSS animations
Extendable color scheme
Every color scheme is less than 50 lines CSS (if you create new one and wanna share, please make a pull request at github)
HTML5 positive
It uses a lot of HTML elements and features e.g. as history API (when you click on menu items, url changes and new data is loaded without refresh, it works for older browsers with hashbang)
Validation
Validation is not used since I wanted to keep these examples simple. Of course I use client and server validation in real projects. If someone would be interested just ask, I would provide some easy validation examples (even combination of server and client validation, which is today somehow neccessary).
